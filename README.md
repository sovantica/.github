# sovantica/.github

Organization-level community health files for [Sovantica](https://github.com/sovantica).

GitHub uses this repository as a fallback for default files across all repositories in the organization.

## What's here

| File | Surface |
|---|---|
| [`profile/README.md`](profile/README.md) | Renders on [github.com/sovantica](https://github.com/sovantica) as the organization "about" page. |
| [`SECURITY.md`](SECURITY.md) | Linked from every repo's Security tab unless the repo has its own. |
| [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md) | Linked from every repo's Community profile unless the repo has its own. |

## Adding more org-level files

This repo can also hold, as `.github/` inherits them organization-wide:

- `CONTRIBUTING.md` — contributor guide
- `SUPPORT.md` — where to get help (Discussions vs. Issues)
- `FUNDING.yml` — GitHub Sponsors / BuyMeACoffee badges
- `ISSUE_TEMPLATE/*.yml` — form-based issue templates
- `PULL_REQUEST_TEMPLATE.md` — PR checklist

See [GitHub's documentation](https://docs.github.com/en/communities/setting-up-your-project-for-healthy-contributions/creating-a-default-community-health-file) for the full list and precedence rules.

## Source of truth

Content for these files is drafted in the Sovantica architecture repo under `company/brand/channels/github/copy/`. Edits land there first, then sync into this repo.
